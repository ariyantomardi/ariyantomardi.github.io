Javascript: Bahasa Curang

  • May 15, 2017
  • post

Bagi sebagian programmer javascript adalah bahasa yang sudah tidak asing lagi khususnya bagi seorang Frontend engineer, javascript sering kali di sebut sebagai nyawa bagi web atau aplikasi karena biasanya javascript di gunakan untuk membuat UI/UX atau interaksi yang mambuat applikasi yang kita gunakan menjadi hidup.

Javascript

Javascript atau JS adalah bahasa yang sering di gunakan seorang FrontEnd Engineer, saya sendiri masih terbiasa menggunakan JS untuk membuat UI/UX dalam sebuah aplikasi/website, JS sendiri menurut saya adalah bahasa pemrograman yg paling gampang di pelajari, saya ingat ada suatu quotes mengatakan:

” JavaScript is the only language that Iâ€™m aware of that people feel they donâ€™t need to learn before they start using it. “ - DOUGLAS CROCKFORD

Bahkan menurut TIOBE (The Importance of Being Earnest) JS berada di nomor 7 pada urutan “20 Best Programming Languages In 2017”, hal itu membuat JS berada di atas GO bahasa pemrograman keluaran google yang berada di nomor 14.

Bahasa yang Curang?

Ya benar anda tidak salah baca “bahasa yang curang”, saya ingat betul ada seorang kolega kerja saya bilang:

” Developer yang masih pakai JS itu ujung-ujungnya cuma jadi agen MLM! “ -Donnie Prakoso

Ya dia bilang seperti itu, dia adalah seorang Backend Engineer yang terbiasa menulis code dengan logic yang rapih, dia kaget ketika tahu JS itu cuma main “curang”. Ya memang saya akui JS itu main “curang” apalagi soal urusan UI/UX, code yang paling sering di gunakan oleh seorang FrontEnd Engineer adalah:


document.getElementById("IdTarget").innerHTML = "hello";  
document.getElementById("IdTarget").style.display = "none"; 

//Atau dalam Jquery:  

$("#idTarget").html("hello");  
$("#idTarget").css("display", none);

Itu adalah code yang sering umum di gunakan untuk membuat interaksi atau tampilan di website, men-display none-kan container yang satu dan men-display block-kan container yang lainya, code itu yang di bilang curang, Apakah ada cara yg lebih baik dari itu? dengan memasukan itu semua kedalam satu function atau di buat variable?

Kalau JS di katakan “Bahasa Curang” atau yang memakai JS di katakan sebagai agen MLM, berarti BRENDAN EICH seorang yang menemukan JS adalah seorang makelar MLM :p

Tapi yang jadi pertanyaan bukankah semua bahasa pemrograman itu hanya main “curang”? :p
